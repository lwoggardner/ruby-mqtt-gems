# Testing Guide

## Local vs Remote Broker Testing

The connection tests can run against either a local mosquitto broker or test.mosquitto.org.

### Local Broker (Default)

Start the local broker:
```bash
docker-compose up -d
```

Run tests:
```bash
bundle exec ruby spec/connection_spec.rb
```

### Remote Broker (test.mosquitto.org)

Run tests against the public test broker:
```bash
TEST_BROKER=test.mosquitto.org bundle exec ruby spec/connection_spec.rb
```

## Broker Configuration Comparison

Both brokers provide similar listeners for comprehensive connection testing:

| Port | Protocol | Auth | Description |
|------|----------|------|-------------|
| 1883 | MQTT | Anonymous | Standard MQTT |
| 1884 | MQTT | Password (ro:readonly) | Authenticated MQTT |
| 8883 | MQTTS | Anonymous | SSL with custom certificate |
| 8884 | MQTTS | Client cert required | Mutual TLS |
| 8885 | MQTTS | Password | SSL + password auth |
| 8886 | MQTTS | Anonymous | SSL with default CA bundle |
| 8887 | MQTTS | Anonymous | SSL with expired cert (error testing) |
| Unix | MQTT | Anonymous | Unix domain socket (local only) |

## SSL Certificates

Local broker uses self-signed certificates generated by `spec/fixture/mosquitto/generate_certs.sh`.

For test.mosquitto.org, use the provided `spec/fixture/ssl/mosquitto.org.crt` CA certificate.

## Running Other Specs

Run client integration tests against local broker:
```bash
bundle exec ruby spec/client_spec.rb
```

The spec_helper.rb configures which brokers to test against via the `with_brokers` method.
